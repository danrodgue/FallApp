NOTA PARA PROFESORES:
Somos conscients de que la base de datos en bruto no debería subirse al repositorio, pero en este caso hemos optado por hacerlo por : 
1) tamaño de la base de datos muy bajo
2) no incluye fotos, ni elementos pesados.
3) conveniente para un despliegue rápido inmediato y automatizado en caso de caída catastrofica del servidor con pérdida de datos. (común en Amazon AWS con máquinas con baja memoria)
